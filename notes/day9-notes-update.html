Linear relationships¶

To say that two random variables from a dataset are linearly related is to say that if we made a scatterplot of the two variables, we could draw a straight line that would approximately 'fit' the data. It is our goal today to understand what 'fit' means in this context.

Consider the following code cell which plots the two variables X and Y. The resulting plot shows that X and Y are linearly related, and the drawn line roughly 'fits' the data.

Since the relationship between the variables is approximately linear, we can 'draw' a straight line that is, on average, equally close to every data point.

To measure the distance from the red line to a data point, we draw a straight line perpendicular to the red line until it reaches the data point. We won't actually draw such lines, the idea of distance from the line to the data is what's important.

Mathematically, we would say that we are trying to 'minimize the sum of the squared errors'. We'll put this into notation in the next section.

Simple linear regression¶

As we discussed on Day 5, simple linear regression assumes that two random variables $X$ and $Y$ are linearly related. In other words, we assume that $$ y \approx \beta_0 + \beta_1x, $$ for all $x$ and $y$ pairs in the respective data columns $X$ and $Y$, and where we use $\approx$ instead of $=$ to account for random variability in the data. Based on the data, we make an estimate of $y$-values that would follow an exactly linear relationship. We define this second linear relationship or model as $$ \hat{y} = \hat{\beta_0} + \hat{\beta_1}x. $$

We use equality in this expression, since this will give us the equation for our line. The ^ (hat or 'carat') symbol represents the fact that these parameters are estimates. We estimate the parameters $\hat{\beta_0}$ and $\hat{\beta_1}$ by minimizing the sum of the squared errors, which is given by $$ \sum (y - \hat{y})^2, $$ where the sum is taken over all $x,y$ pairs in the columns $X$ and $Y$. We often see this called the 'residual sum of squares' or 'RSS', and so we'll refer to it by RSS. If you want to find the minimum, you can take the derivative, set it to zero, and solve for the minimizer (but let's allow Python to do that for us).

We will also refer to the total sum of squares or TSS, which is defined by $$ TSS = \sum (y - \bar{y})^2, $$ where $\bar{y}$ is the mean of the $y$-values. Notice that this is almost exactly the expression for the variance of the $y$ variable (the variance takes the extra step of dividing by the number of observations).

Testing the model¶

We first test the parameter $\beta_1$ for statistical significance. Only the $\beta_1$ parameter needs to be tested, since it is the only parameter involving $X$. Expressed as a formal hypothesis test, this test can be written as $$ H_0 : \beta_1 = 0, \quad H_a : \beta_1 \neq 0. $$

The actual testing is done by checking the $t$ statistic for the $\hat{\beta}$ parameter, which comes with a corresponding confidence interval and a $p$-value. The $p$-value is the probability of seeing the given $t$-statistic, and the parameter is usually said to be significant if $p < 0.05$ (95% confidence). Note that the intercept may similarly be tested for significance.

Next, we can measure the proportion of variability accounted for by our model. This proportion is calculated by the following ratio: $$ \frac{TSS - RSS}{TSS} =: R^2. $$

Higher values of $R^2$ correspond to a higher percentage of variability being explained by our model ($0 \leq R^2 \leq 1$).

Let's take a look at an example and see where these numbers are displayed in the output. We have two options for fitting linear regression models: sklearn and statsmodels. We'll use both in the example.

Interpreting the regression output¶

The summary() method for the ols object provides us with a wealth of information about our model. On the upper left, we see basic information such as the dependent variable, the model and method for fitting, the date and time, and the degrees of freedom.

On the upper right, we see the $R^2$ value discussed above. Immediately below $R^2$ is a value called Adj. R-squared, sometimes denoted as $R_a^2$. This value accounts for the number of predictor variables in the model. It has been discovered that $R^2$ will automatically increase when more predictors are added to the model, and $R_a^2$ corrects for this phenomenon. Often the $R_a^2$ value is reported along with $R^2$, as it is a less biased measure of the amount of variability accounted for by the model.

Below $R_a^2$, we see the F-statistic and Prob (F-statistic). We saw this statistic when we talked about ANOVA on Day 4. The F-statistic can be described by: $$ F = \frac{\text{explained variance}}{\text{unexplained variance}}. $$ In other words, when the F-statistic is large, the model explains more of the variance in the data. The corresponding $p$-value for the F-statistic is found in Prob (F-statistic). We say that the model is statistically significant if Prob (F-statistic) < 0.05.

Technical outputs¶

Next on the upper right of the output are the Log-Likelihood, AIC, and BIC. These are much more technical in nature, but can be simply summarized as follows. The Log-Likelihood is the quantity maximized during maximum likelihood estimation. Higher values are better.

The Akaike Information Criterion, or AIC, is used when selecting which predictor variables are important for the model. When AIC is lower, that means the model is getting closer to representing the truth. Similarly, when the Bayesian Information Criterion, or BIC, is lower, then the model is getting closer to representing the truth.

For further reading on this, a good place to start is Wikipedia.

Model parameters¶

After these statistics we see tabular output for Intercept and X. The coef column contains the coefficients or parameter values for $\hat{\beta_0}$ and $\hat{\beta_1}$, respectively (recalling that our model was $\hat{y} = \hat{\beta_0} + \hat{\beta_1}x$, where $\hat{\beta_0}$ is the intercept and $\hat{\beta_1}$ is the slope of the regression line).

These parameters both come with the standard error (std err) and $t$-statistics with their corresponding $p$-values. The last two columns [0.025 and 0.975] represent the respective lower and upper bounds of the 95% confidence interval for the $t$-statistic.

Other statistics¶

Next, we see the Omnibus statistic with its corresponding $p$-value Prob(Omnibus). This tests the normality of the residuals, which are defined by the prediction error in the predicted $y$ values. The Skew and Kurtosis parameters are the fitted parameters for a normal distribution.

The Durbin-Watson statistic is mainly used in time series, and won't be discussed here. The Jarque-Bera (JB) statistic is another test for the normality of the residuals, and comes with its $p$-value Prob(JB).

Finally, the Cond. No. is the condition number of the $X^TX$ matrix. Further implications of this number won't be discussed here.

Multiple linear regression¶

Multiple regression involves using more than one predictor variable (often called features) to estimate the target variable $Y$. All of the above information for simple linear regression remains for multiple regression, only now we assume the expanded linear relationship $$ y \approx \beta_0 + \beta_1x_1 + \beta_2x_2 + \cdots + \beta_px_p, $$ where $X = \{ x_1,x_2,\ldots,x_p \}$ represent the columns of predictor variables. We then fit the model $$ \hat{y} = \hat{\beta_0} + \hat{\beta_1}x_1 + \hat{\beta_2}x_2 + \cdots + \hat{\beta_p}x_p. $$

Selecting important parameters¶

When dealing with possibly many predictor variables, it would be very useful to know which variables matter to the model and which ones could be ignored. There are two main methods for variable selection: forward selection and backward elimination.

Forward selection recursively adds one predictor variable into the model and keeps only variables that improve the $R_a^2$ value. An implementation by Trevor Smith and Aaron Schumacher is used in the example below.

Backward elimination follows the same principles as forward selection, but starts with all predictors and eliminates recursively. Both techniques are often collectively referred to as stepwise regression.

def forward_selected(data, response):
    """Linear model designed by forward selection.

    Parameters:
    -----------
    data : pandas DataFrame with all possible predictors and response

    response: string, name of response column in data

    Returns:
    --------
    model: an "optimal" fitted statsmodels linear model
           with an intercept
           selected by forward selection
           evaluated by adjusted R-squared
    """
    remaining = set(data.columns)
    remaining.remove(response)
    selected = []
    current_score, best_new_score = 0.0, 0.0
    while remaining and current_score == best_new_score:
        scores_with_candidates = []
        for candidate in remaining:
            formula = "{} ~ {} + 1".format(response,
                                           ' + '.join(selected + [candidate]))
            score = ols(formula, data).fit().rsquared_adj
            scores_with_candidates.append((score, candidate))
        scores_with_candidates.sort()
        best_new_score, best_candidate = scores_with_candidates.pop()
        if current_score < best_new_score:
            remaining.remove(best_candidate)
            selected.append(best_candidate)
            current_score = best_new_score
    formula = "{} ~ {} + 1".format(response,
                                   ' + '.join(selected))
    model = ols(formula, data).fit()
    return model

print(forward_selected(train, 'Y').summary())

In this case, all predictors except SOP were found to be significant in the model.

Interpreting the parameters for predictor variables¶

The huge benefit of a significant linear model is that the $\beta$ parameters have actual meaning. In particular, the value of a given $\beta_i$ parameter represents the predicted contributed increase in $y$ for a unit increase of $x_i$. For simple linear regression, it is easy to see how different values of $\beta_0$ and $\beta_1$ affect the model, since the resulting model can be plotted as a line.

However, in multiple regression it is more difficult to imagine the plot of the resulting model, since the model represents a hyperplane over $p$ dimensions (where $p$ is the number of independent predictors). Still, if we restrict our attention to a single predictor variable, we can plot its influence over the response variable $y$ with ease. These plots can be done using the sm.graphics.plot_partregress_grid(<model>) function from statsmodels.

Problems arising in regression¶

Many potential problems arise in linear regression, and a large body of information, tips, and tricks has been created as a result. Rather than listing all possible problems and their candidate solutions, we'll just look at 4 problems identified in (James, Witten, Hastie, & Tibshirani, 2013).

  1. The data shows non-linear relationships - it may be best to consider a non-linear model to describe the relationships in the data.
  2. The errors in the prediction have non-constant variance - this shows up when the variance of a single error term depends on the value of the response variable $Y$. This is sometimes combated by transforming the response variable with logarithms or square root functions. Another solution is to use weighted least squares.
  3. Outliers - if there is some mistake in the recorded value or the absence of the outlier doesn't significantly affect $R_a^2$, it is likely safe to remove the value. Sometimes, however, an outlier should stay in the dataset and will significantly affect the fit of the model. This leads to an investigation of leverage statistics, and further reading in (James, Witten, Hastie, & Tibshirani, 2013) is recommended.
  4. Two or more predictor variables are related to one another - this can be detected by examining the covariance matrix for all predictor variables. If an entry of this matrix has a large absolute value, then the corresponding variables are related. We can work to solve this problem using PCA with all predictor variables and then fit the model on the de-correlated principal components.
